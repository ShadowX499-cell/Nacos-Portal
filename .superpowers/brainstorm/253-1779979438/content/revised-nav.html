🏫 School Fees NEW PAGE
Dedicated page for paying school fees via Paystack. Shows payment status per session (Paid / Unpaid). Amount set by admin. Generates receipt on payment.

Backend: Uses existing PaymentType.school_fees enum — same Paystack flow as result subscription. Admin sets amount via env var or config.
📝 Course Registration NEW PAGE + SCHEMA
Student uploads their course registration form (PDF/image) each semester. Admin reviews and marks it verified or rejected with a note. Student sees current status: Pending · Verified ✓ · Rejected ✗.

Backend: Needs a new CourseRegistration Prisma model (userId, session, semester, fileUrl, status, reviewNote). File stored on AWS S3 (already planned for Phase 2+). Admin review endpoint added to admin module.
⚠️ Requires 1 new Prisma model + S3 file upload. Adds meaningful backend scope.
🔔 Notifications / Upcoming Events SCHEMA EXISTS
Bell icon in topbar shows unread count. Dedicated Notifications page lists all department announcements, result publication alerts, election notices, and payment reminders — sorted newest first.

Backend: Notification + NotificationRead models already in schema. Just needs GET /api/v1/notifications (list) and POST /api/v1/notifications/:id/read (mark read) endpoints. Also surfaced on Dashboard in the Action Required panel.
💳 Payments (Result Subscriptions) EXISTING
Unchanged — shows history of result subscription payments. School Fees moved to its own dedicated page above.

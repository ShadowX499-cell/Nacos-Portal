Design — Section 1 of 4: Database Schema Changes

All schema additions are additive (no breaking changes). Review each model change below.

User model — new fields

// existing fields stay unchanged
profilePhotoUrl String? @map("profile_photo_url")
dateOfBirth DateTime? @map("date_of_birth") @db.Date
stateOfOrigin String? @map("state_of_origin") @db.VarChar(50)
lga String? @db.VarChar(100)
homeAddress String? @map("home_address") @db.VarChar(300)
superAdminType SuperAdminType? @map("super_admin_type")
// userId format: NACOS/CSC/2024/XXXXX (5-digit random, not sequential)

New enums

enum SuperAdminType {
  course_adviser
  hod
  result_exam_officer
}

enum StudentStatus {
  active     // normal enrolled student
  carryover  // has ≥1 failed course from a published result
  graduated  // was L400, session advanced
  suspended  // existing — manually set by HOD
}

⚠️ Note: The existing UserStatus enum (pending/validated/suspended) stays for account activation. StudentStatus is a new separate field on User tracking academic standing.

StudentGrade model — carryover flag

// existing fields stay unchanged
isCarryover Boolean @default(false) @map("is_carryover")
carriedFromSession String? @map("carried_from_session") @db.VarChar(20)
resolvedAt DateTime? @map("resolved_at") // set when student later passes

AcademicTransition model — new

// audit trail for every semester/session advance
model AcademicTransition {
  id String @id @default(uuid())
  departmentId String
  fromSession String @db.VarChar(20)
  fromSemester Semester
  toSession String @db.VarChar(20)
  toSemester Semester
  studentsAdvanced Int
  studentsGraduated Int
  initiatedById String @map("initiated_by")
  initiatedAt DateTime @default(now())
}

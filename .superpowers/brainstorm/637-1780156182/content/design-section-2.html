Design — Section 2 of 4: API & Service Layer

New endpoints grouped by sub-project. All follow existing conventions: asyncHandler, response helpers, audit log on every mutation.

Sub-project E — Profile & Registration

POST /api/v1/admin/users               ← multipart/form-data (adds photo + bio fields)
PATCH /api/v1/admin/users/:id        ← update bio fields, replace photo
GET    /api/v1/admin/users?program=CSC ← filter by program (new query param)
GET    /api/v1/uploads/:filename     ← Express static served from uploads/
GET    /api/v1/profile/me           ← extended to include new bio fields

ID generation: NACOS/[PROGRAM]/[YEAR]/[5-digit random] — uses crypto.randomInt(10000, 99999) with collision retry (max 5 attempts).

Sub-project F — Super Admin RBAC & Audit Logs

POST /api/v1/admin/super-admins       ← HOD only: create course_adviser / result_exam_officer
GET    /api/v1/admin/super-admins       ← list all super admins + their role
DELETE /api/v1/admin/super-admins/:id   ← HOD only: revoke super admin

GET    /api/v1/admin/audit-logs        ← HOD only, filters: actorId, role, action, entity, dateFrom, dateTo
GET    /api/v1/admin/audit-logs/me    ← any super admin: own trail only
GET    /api/v1/admin/audit-logs/:id   ← full before/after diff for one entry

Permission middleware: requirePermission('publish_results') — a thin wrapper that reads role + superAdminType against a static permission map. Used on every protected route.

Sub-project D — Academic Enrollment

GET    /api/v1/admin/dashboard          ← adds studentsByProgram[] to existing response
GET    /api/v1/admin/users?program=ICT   ← same endpoint, new filter param

POST /api/v1/admin/academic/advance   ← HOD only: advance semester or session
                                       body: { type: "semester" | "session", confirm: true }
GET    /api/v1/admin/academic/transitions ← history of past advances
GET    /api/v1/admin/academic/preview   ← HOD: dry-run showing how many students will advance/graduate

GET    /api/v1/student/carryovers       ← student: list own carry-over courses

Publish-results side effect: When a gradebook is published, the results service scans every StudentGrade in that gradebook. Any grade = F sets isCarryover = true on that row and updates the student's studentStatus to carryover. If a student later passes the course in a subsequent gradebook, resolvedAt is stamped and status recalculated.

Sub-project G — Revenue Dashboard

GET /api/v1/admin/revenue                ← HOD only
    query: dateFrom, dateTo, session, semester,
          program, level, type (nacos_dues|school_fees|…)
    returns: { totalRevenue, byType[], byProgram[], byLevel[],
               monthlyTrend[], recentPayments[], collectionRate }

GET /api/v1/admin/revenue/export        ← CSV download, same filters

Sub-project H — Student Dashboard Badges (small)

GET /api/v1/student/compliance         ← new endpoint
    returns: { nacosDue: { paid: 2, total: 8 },
               schoolFees: { paid: 1, total: 8 },
               courseForm: { verified: 1, total: 8 } }
